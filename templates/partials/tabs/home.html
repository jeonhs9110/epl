FOBO AI

Football Outcome Prediction Optimiser 축구 경기 결과 베팅 최적화 AI

A multi-stage machine-learning pipeline that scrapes 13 European leagues, fuses transformer + graph-attention + gradient boosting models, and uses reinforcement learning to decide when a prediction carries enough statistical edge to act on. 유럽 13개 리그의 경기 데이터를 수집하고, 트랜스포머·그래프 어텐션·그래디언트 부스팅 모델을 융합한 뒤, 강화학습으로 수학적으로 베팅이 합리적인 순간만 선별하는 다단계 머신러닝 파이프라인입니다.

① System Overview시스템 개요

FOBO AI ingests historical results from 13 European leagues, trains a hybrid deep-learning + tree-based ensemble, and applies a reinforcement-learning agent to decide when a prediction carries enough statistical edge to act on. The final output is served via this web dashboard.

FOBO AI는 유럽 13개 리그의 경기 기록을 수집해 딥러닝·트리 앙상블을 학습시키고, 강화학습 에이전트가 수학적으로 합리적인 베팅 순간을 판단합니다. 최종 결과는 본 웹 대시보드를 통해 제공됩니다.

Why multi-league?왜 여러 리그인가?

A single cross-league model learns shared dynamics (home advantage, goal distributions) while league-specific embeddings capture tactical differences. Cross-league transfer generalises better to low-data leagues like the Championship.

하나의 범리그 모델은 홈 어드밴티지·득점 분포 같은 공통 요소를 학습하고, 리그별 임베딩이 전술적 차이를 포착합니다. 덕분에 Championship처럼 데이터가 적은 리그에서도 일반화 성능이 향상됩니다.

③ Deep Learning — LeagueAwareModel딥러닝 모델 — LeagueAwareModel
Why a Transformer for match sequences?경기 시퀀스에 트랜스포머를 쓰는 이유

A team's last N matches form a sequence where order matters — a 5-game winning streak differs from 5 scattered wins. A Temporal Transformer Encoder with Learnable Positional Encoding captures ordering, and attention pooling extracts the most informative matches rather than averaging blindly.

팀의 최근 N경기는 순서가 중요한 시퀀스입니다 — 5연승과 산발적 5승은 성격이 다릅니다. 학습 가능한 위치 인코딩을 가진 시간적 트랜스포머 인코더가 순서를 포착하고, 어텐션 풀링이 단순 평균이 아닌 가장 유의미한 경기를 추출합니다.

Graph Attention Network (GAT)그래프 어텐션 네트워크 (GAT)

Football is relational — a team's strength is partly revealed by who they beat. The GAT propagates embeddings across a match graph so that beating a strong opponent counts more than beating a weak one. Flat embeddings can't capture this.

축구는 관계적인 스포츠입니다 — 누구를 이겼는지가 실력의 일부를 보여줍니다. GAT는 경기 그래프를 통해 임베딩을 전파해 강팀을 이긴 경우가 약팀을 이긴 경우보다 더 큰 가중치를 갖도록 합니다. 평면 임베딩으로는 담을 수 없는 정보입니다.

Dixon-Coles LossDixon-Coles 손실함수

The model outputs Poisson rate parameters λ_home and λ_away (expected goals). Win/Draw/Loss probabilities are then derived analytically from the Poisson distribution — not predicted directly. This follows Dixon-Coles (1997), the academic gold standard for football prediction. The Dixon-Coles correction adjusts low-scoring results (0-0, 1-0, 0-1, 1-1) that naive Poisson systematically mispredicts. Draw outcomes are upweighted 1.5× to counter home-win bias.

모델은 λ_home, λ_away (기대 골 수)를 출력합니다. 승/무/패 확률은 Poisson 분포에서 해석적으로 유도되며, 이는 축구 예측의 학계 표준인 Dixon-Coles(1997) 모델을 따릅니다. Dixon-Coles 보정은 단순 Poisson이 체계적으로 틀리는 저득점 결과 (0-0, 1-0, 0-1, 1-1)를 교정합니다. 홈팀 편향을 상쇄하기 위해 무승부 결과에는 1.5배의 가중치를 둡니다.

④ Reinforcement Learning — PPO Decision Agent강화학습 — PPO 베팅 에이전트

Prediction accuracy alone doesn't guarantee a positive decision. A 55% home-win prediction is only worth acting on if the market's implied probability is under 55%. The PPO agent learns this decision boundary directly from simulated outcome feedback.

예측 정확도만으로 수익이 보장되지 않습니다. 55% 홈 승리 예측도 북메이커의 내재 확률이 55% 미만일 때만 수익성이 있습니다. PPO 에이전트는 시뮬레이션된 손익 피드백을 통해 이 의사결정 경계를 학습합니다.

Why PPO?왜 PPO인가?

PPO is the industry standard for discrete RL — stable, sample-efficient, and resistant to the catastrophic policy collapses of vanilla policy gradient. The clipped surrogate objective prevents overly large updates:

PPO는 이산 강화학습의 업계 표준입니다 — 안정적이고 샘플 효율적이며, 기본 정책 경사법의 붕괴 문제에 강합니다. 클립된 대리 목적함수가 과도한 업데이트를 방지합니다: 

L_CLIP = E[ min( r_t · A_t, clip(r_t, 1−ε, 1+ε) · A_t ) ]   (ε = 0.2)
Actions & Kelly-Criterion Reward행동·Kelly 기준 보상

The agent picks one of 4 actions per match: 0 Home, 1 Draw, 2 Away, 3 Pass. Rewards are shaped by the Kelly Criterion — a mathematically optimal position sizing formula that maximises long-run growth under an uncertain probability model:

에이전트는 경기마다 4가지 행동 중 하나를 선택합니다: 0 홈 베팅, 1 무승부 베팅, 2 원정 베팅, 3 베팅 스킵. 보상은 장기 자본 성장을 최대화하는 수학적 최적 베팅 크기 공식인 Kelly 기준을 따릅니다: 

f* = (b·p − q) / b
  b = odds − 1,  p = model win prob,  q = 1 − p

If Kelly ≤ 0, expected value is negative and passing is rewarded. Correct picks earn odds-proportional reward (capped at 3× to prevent longshot-chasing behaviour). Incorrect picks incur a −2 penalty to discourage low-edge decisions.

Kelly ≤ 0이면 기대값이 음수이므로 베팅 스킵에 보상을 주고, 베팅 성공 시에는 배당에 비례하는 보상(3배 한도, 장기 무리베팅 방지)을 줍니다. 실패 시 −2 페널티로 가치 없는 베팅을 억제합니다.

⑤ Hybrid Ensemble — XGBoost + LightGBM하이브리드 앙상블 — XGBoost + LightGBM

Neural networks and gradient-boosted trees make different kinds of errors. The Transformer captures long-range temporal patterns and team-graph interactions, while XGBoost/LightGBM excel at sharp tabular boundaries ('team has not scored in 3 home games'). Averaging all three gives consistently better calibration than any single model.

신경망과 그래디언트 부스팅 트리는 오류 양상이 다릅니다. 트랜스포머는 장기 시계열 패턴과 팀 그래프 상호작용을 포착하고, XGBoost/LightGBM은 '홈 3경기 무득점' 같은 날카로운 표 형태의 경계에서 강합니다. 세 모델의 확률을 평균내면 단일 모델보다 일관되게 더 잘 보정된 결과를 얻을 수 있습니다.

XGBoost and LightGBM are trained on 2089-dim DL embeddings, not raw data. This is transfer learning — the trees inherit the Transformer's rich representations. Platt scaling then calibrates XGBoost's raw probabilities (a predicted 70% often reflects less certainty than stated) so that Kelly-Criterion sizing stays reliable.

XGBoost·LightGBM은 원본이 아니라 2089차원 딥러닝 임베딩으로 학습합니다. 이는 전이학습으로, 트리 모델이 트랜스포머의 표현력을 물려받습니다. 이후 Platt 스케일링이 XGBoost의 원시 확률(예측된 70%가 실제로는 그보다 낮은 확신일 때가 많음)을 보정해 Kelly 크기 산정이 안정적으로 작동하게 합니다.

  2. Retrieve recent match history sequences from the master DataFrame
  3. Build GNN adjacency matrix for the match context
  4. Run LeagueAwareModel forward pass → λ_home, λ_away, ρ

  6. Extract 2080-dim feature vector → run XGBoost + calibrator + LightGBM
  7. Average all model probabilities → ensemble W/D/L
  8. Pass state to PPO agent → recommended action + Kelly fraction
  9. Return JSON with probabilities, score, xG, BTTS, Over 2.5, and suggested pick

⑦ Design Philosophy설계 철학
Domain-driven loss도메인 기반 손실함수

Dixon-Coles NLL forces the model to reason about goals, not outcomes. Better-calibrated probabilities, honors the math of football scoring.

Dixon-Coles NLL은 모델이 결과가 아닌 골 수를 추론하게 만듭니다. 확률 보정이 개선되고, 축구 득점의 수학 구조를 존중합니다.

Prediction vs decision예측과 결정의 분리

DL answers "what will happen"; PPO answers "should I act". Separating these lets each be trained with the right loss signal.

DL은 "무엇이 일어날까"에, PPO는 "베팅해야 하나"에 답합니다. 분리함으로써 각각 올바른 손실 신호로 학습됩니다.

GPU-first designGPU 우선 설계

All heavy compute (Transformer, GAT, PPO) auto-routes to CUDA. Batch 64 (CPU) → 512 (GPU), 8× more data-efficient per wall-clock minute.

트랜스포머·GAT·PPO의 모든 무거운 연산은 자동으로 CUDA로 라우팅됩니다. 배치 64(CPU) → 512(GPU)로 벽시계 시간당 8배 더 효율적입니다.

Resilient pipeline복원력 있는 파이프라인

Every expensive artifact (DL, PPO, XGBoost, calibrator) is saved immediately after training. Pipeline resumable at any stage — no lost compute.

DL·PPO·XGBoost·보정기 등 모든 비싼 산출물은 학습 직후 저장됩니다. 파이프라인은 어느 단계에서든 재개 가능 — 연산 손실이 없습니다.
